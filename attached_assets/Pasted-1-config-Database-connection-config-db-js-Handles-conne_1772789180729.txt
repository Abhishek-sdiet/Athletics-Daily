1️⃣ config

Database connection.

config/db.js

Handles connection to MongoDB.

2️⃣ models (Database Schema)
User.js

Stores player accounts.

User
- _id
- username
- email
- password
- createdAt
Question.js

Stores daily puzzles.

Question
- _id
- questionText
- answer
- date
- category
- createdAt

Example:

{
questionText: "City of 2012 Olympic athletics",
answer: "LONDON",
date: "2026-03-10"
}

This allows you to upload 7 days questions easily.

GameResult.js

Stores player results.

GameResult
- userId
- questionDate
- attempts
- isSolved
- playedAt

Example:

{
userId: 12345,
questionDate: "2026-03-10",
attempts: 3,
isSolved: true
}

This prevents playing twice.

🧩 Backend Flow
Step 1

Frontend calls

GET /api/game/today

Backend returns:

{
question:"City of 2012 Olympic athletics",
length:6
}

⚠️ Answer NOT sent.

Step 2

User submits guess.

POST /api/game/submit

Body:

{
guess:"LONDER"
}

Backend returns evaluation:

[
"correct",
"correct",
"correct",
"present",
"absent",
"absent"
]
Step 3

When user wins.

Store result:

GameResult
attempts:3
date:2026-03-10
Step 4

Prevent replay.

Before starting game:

Check GameResult where
userId + date

If exists:

"You already played today"
📦 Admin Weekly Upload

You upload 7 puzzles in one go.

Request:

POST /api/admin/upload-week

Body:

🔐 Anti-Cheat Strategy

Important.

Answer should never go to frontend.

Instead:

Frontend sends guess → Backend checks → Backend returns result.
